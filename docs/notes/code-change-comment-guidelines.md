# 代码增删改注释规范

本项目在学习和修改代码时，采用“变更标记型注释”记录代码的新增、修改和删除操作，方便后续根据修改人、日期和原因追溯修改记录。

## 1. 基本原则

每条变更注释应包含以下信息：

- 变更类型：`add`、`modified`、`del`
- 修改人：例如 `gq`
- 修改日期：建议使用 `YYYY-MM-DD` 格式
- 修改原因：简要说明为什么新增、修改或删除

通用格式：

```text
<注释符> <变更类型> by <修改人> [<日期>：<修改原因>]
```

示例：

```text
# add by gq [2026-04-28：增加报告合并流程]
```

## 2. 新增代码注释规范

### 2.1 多行新增

新增一整段代码时，在代码块前后分别添加开始和结束标记。

C/C++ 示例：

```cpp
// add by gq [2026-04-28：增加加法函数]
int add(int a, int b)
{
    a += 1;
    b += 1;

    return a + b;
}
// add end
```

Python 示例：

```python
# add by gq [2026-04-28：增加加法函数]
def add(a, b):
    a += 1
    b += 1

    return a + b
# add end
```

### 2.2 单行新增

只新增一行代码时，可以在行尾标记。

C/C++ 示例：

```cpp
int c = a + b; // add by gq [2026-04-28：增加中间结果变量]
```

Python 示例：

```python
c = a + b  # add by gq [2026-04-28：增加中间结果变量]
```

## 3. 修改代码注释规范

### 3.1 多行修改

修改一整段代码时，在修改后的代码块前后添加开始和结束标记。

C/C++ 示例：

```cpp
// modified by gq [2026-04-28：调整参数递增逻辑]
int add1(int a, int b)
{
    a += 2;
    b += 3;

    return a + b;
}
// mod end
```

Python 示例：

```python
# modified by gq [2026-04-28：调整参数递增逻辑]
def add1(a, b):
    a += 2
    b += 3

    return a + b
# mod end
```

### 3.2 单行修改

只修改一行代码时，可以在行尾标记。

C/C++ 示例：

```cpp
return c; // modified by gq [2026-04-28：改为返回预先计算结果]
```

Python 示例：

```python
return c  # modified by gq [2026-04-28：改为返回预先计算结果]
```

## 4. 删除代码注释规范

删除代码时，学习阶段可以先不物理删除，而是将原代码注释掉，并添加删除标记，方便之后查看历史逻辑。

### 4.1 多行删除

C/C++ 示例：

```cpp
// del by gq [2026-04-28：删除旧版加法逻辑]
// int add1(int a, int b)
// {
//     a += 2;
//     b += 3;
//
//     return a + b;
// }
// del end
```

Python 示例：

```python
# del by gq [2026-04-28：删除旧版加法逻辑]
# def add1(a, b):
#     a += 2
#     b += 3
#
#     return a + b
# del end
```

### 4.2 单行删除

C/C++ 示例：

```cpp
// a += 2; // del by gq [2026-04-28：取消额外自增逻辑]
```

Python 示例：

```python
# a += 2  # del by gq [2026-04-28：取消额外自增逻辑]
```

## 5. 推荐执行规则

1. 修改日期统一使用 `YYYY-MM-DD` 格式。
2. 修改原因要简短明确，重点说明“为什么改”，不要只写“修改代码”。
3. 多行变更要成对出现结束标记：
   - 新增代码使用 `add end`
   - 修改代码使用 `mod end`
   - 删除代码使用 `del end`
4. Python 文件使用 `#` 注释。
5. C/C++、Java、JavaScript 等文件可以使用 `//` 注释。
6. 对学习阶段的代码，可以保留删除代码的注释块，方便回看旧逻辑。
7. 如果以后项目使用 Git 管理，长期维护时可以结合 Git 历史，减少大量过期注释代码。

- 默认日期格式：`YYYY-MM-DD`
