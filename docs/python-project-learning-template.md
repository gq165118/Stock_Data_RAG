# Python 项目代码学习模板
用于通过当前项目学习 Python 代码，默认面向已有 C++ 开发经验的学习者。
## 讲解模板
每次讲解一小段代码时，按以下结构进行：
1. **代码片段**
   - 只选取当前要理解的关键代码，不一次性展开太多。
2. **Python 语法解释**
   - 解释当前代码涉及的 Python 语法、库用法或惯用写法。
   - 优先讲当前代码需要理解的内容，不展开过多无关理论。
3. **C++ 类比**
   - 用 C++ 中相近的概念辅助理解。
   - 同时指出 Python 与 C++ 的关键差异。
4. **项目中的作用**
   - 说明这段代码在当前项目流程中的职责。
   - 尽量解释它和其他文件、类、函数的调用关系。
5. **关键词**
   - 列出需要记住的核心术语，例如 `import`、`def`、`decorator`、`class`、`self`、`Path`、`dict` 等。
6. **常见误区**
   - 点出 Python 初学者，尤其是 C++ 背景学习者容易误解的地方。
7. **小练习**
   - 给一个很小的阅读、修改、运行或复述任务。
   - 练习应尽量能在当前项目代码中完成。
8. **知识点选择题**
   - 用 1 到 3 道选择题巩固当前知识点。
   - 每题给出选项和答案，必要时附一句解释。
9. **下一步追踪**
   - 指出下一步应该继续阅读哪个函数、类、文件或调用链。
## 使用原则
- 一次只讲一小段代码，避免大段灌输。
- 优先围绕当前项目代码解释，而不是脱离项目讲 Python 教程。
- 对已有 C++ 基础的概念可以简讲，对 Python 特有语法重点讲。
- 遇到复杂调用链时，先画流程，再深入细节。
- 模板可以随着学习过程不断调整和精简。
## 当前推荐学习顺序
1. `main.py`
   - 命令行入口、`import`、函数、装饰器、`click`、`Path`、对象创建、配置字典、程序入口。
2. `src/pipeline.py`
   - `class`、`__init__`、`self`、方法调用、流程编排、配置传递。
3. `src/questions_processing.py`
   - 数据结构、循环、条件判断、问题处理流程。
4. `src/api_requests.py`
   - API 请求、JSON、异常处理、环境变量、网络调用。
